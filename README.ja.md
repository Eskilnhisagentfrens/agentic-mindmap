# Agentic Mindmap

[English](./README.md) · [中文](./README.zh.md) · **日本語**

LLM エージェントと連携できる、ローカル動作の XMind 風マインドマップ。デスクトップ（macOS Electron）とブラウザの両方で動き、**データは完全にあなたのマシン内にとどまります**。

> 🤖 **AI 展開**：任意のノードを選択し、ツールバーの 🤖 を押すと、モデルが自動的にノードの種類を判別し、最適な分解方法を選び、複雑度に応じて深さ（1〜3 階層）を決定します。生成された各子ノードには「なぜそれが妥当な子ノードなのか」を一文で説明する why が付きます — キャンバス上に直接表示されるので、メモパネルを開く必要はありません。

## インストール

### リリース版をダウンロード（ビルド不要）

[**最新リリース**](https://github.com/Eskilnhisagentfrens/agentic-mindmap/releases/latest) — Apple Silicon と Intel Mac 用の DMG が用意されています。

```bash
# 初回起動には Gatekeeper 回避が必要です（DMG は未署名）：
xattr -cr "/Applications/Agentic Mindmap.app"
```

### ソースからビルド（開発者向け）

```bash
git clone https://github.com/Eskilnhisagentfrens/agentic-mindmap.git
cd agentic-mindmap
npm install      # 初回のみ
npm start        # 起動
npm run dist     # dist/ に .dmg を生成
```

### ブラウザ版

`index.html` をダブルクリック、または：

```bash
open index.html
```

スマホで見る場合は AirDrop / iCloud Drive で `index.html` を端末に送り、Safari で開いてください。iPhone 17 Pro のダイナミックアイランドとノッチに最適化されています。

## AI 展開のセットアップ

🤖 ボタンを使うには API キーが必要です — DeepSeek（Anthropic より約 30 倍安く、推奨）か Anthropic を選べます。

**推奨：macOS Keychain（シェル履歴に痕跡を残しません）**

```bash
security add-generic-password -a "$USER" -s "DEEPSEEK_API_KEY" -w 'sk-...'
```

DeepSeek キー登録：https://platform.deepseek.com（国際版）。Anthropic：https://console.anthropic.com。

**代替：環境変数**（`~/.zshrc` などに記述）：

```bash
export DEEPSEEK_API_KEY="$(security find-generic-password -a "$USER" -s DEEPSEEK_API_KEY -w)"
# または：
export ANTHROPIC_API_KEY=sk-ant-...
```

アプリは Electron メインプロセス内で**遅延ロード**でキーを読み込みます。`localStorage`、レンダラのメモリ、シェル履歴のいずれにも到達しません。詳しいセキュリティモデルは [docs/architecture.md](./docs/architecture.md) を参照してください。

## AI 展開の仕組み

ノードの 🤖 ボタンをクリックすると：

1. モデルがまず**ノードの種類を判別**します — goal（目標）/ concept（概念）/ question（疑問）/ option（選択肢）/ process（プロセス）/ artifact（成果物）。
2. 次に**複雑度を判断して深さ（1〜3 階層）を選択**します — 「ルーター再起動」のような原子的タスクは深さ 1、「SaaS 製品の構築」のような多段階プロジェクトは深さ 3（最大 40 ノード）まで展開。**枝のテーパー化**で暴走を防ぎます（トップ 3-6、中段 2-4、最下段 2-3）。
3. ノードの種類に応じた**適切な分解方法**を適用 — 目標には動詞始まりのタスク、トピックにはサブ概念、選択肢には比較軸など。
4. **兄弟ノード認識**：同じ深さの既存ピアがモデルに渡されるため、生成される子ノードは周囲のツリーのスタイルと粒度に合わせられます。
5. 各子ノードには**複数文の why** が付き、その役割を説明 — 最初の約 3 行はキャンバス上の見出し下のサブタイトルとしてインライン表示（青い点インジケータ付き）。📝 で全文表示。
6. プロンプトは**実体名**（実在する企業・規制・製品）、**意思決定の根拠となる数値**（価格・期限・市場規模）、**具体的な推奨**を明示的に要求 — 「市場分析」のような汎用カテゴリは明示的な失敗モードとされています。

パルス動作する 🤖 のプログレスオーバーレイは、リアルタイム経過秒数、4 段階のステータステキスト、漸近的なプログレスバー（応答が実際に到着するまで 100% に到達しないため、「99% で固まる」感覚がない）を表示します。

### マウス / タッチ

- **ノードのドラッグ**：
  - 他ノードの中央にドロップ → 子ノードになる
  - 他ノードの上端 / 下端にドロップ → 前 / 後ろの兄弟になる
  - 空白にドロップ → 自由位置（緑の点線枠が出て、サブツリーごと移動）
  - ドラッグ中に `Esc` でキャンセル
- **ノードのリサイズ**（選択するとハンドル表示）：
  - 右下の丸印 → 幅と高さを自由に変更
  - 右辺 → 幅のみ（テキスト自動折返し）
  - 下辺 → 高さのみ
  - `Shift` + 右下 → 比率維持
- **キャンバス**：
  - スクロール / 2 本指トラックパッド → パン
  - `⌘` + スクロール / ピンチ → ズーム
  - 空白部分のドラッグ → キャンバスをパン

### iPhone

- タップで選択、ダブルタップ / 長押しで編集
- 下部ツールバー：子追加、兄弟追加、アイコン、色、メモ、削除
- ピンチでズーム、シングルタッチでパン
- それ以外は上部ツールバーから

## 高度な機能

### 色とアイコン

ツールバー 🎨 で 12 色パレットからノードに色を付けます。**子ノードは最も近い色付き祖先の色を継承**し、エッジの色も枝に追従します。子ノードに別の色を付けたい場合は明示的に色を指定すると上書きされます。

ツールバー 😀 で絵文字アイコン（24 個）を付けられます。

### メモ + キャンバス内インラインプレビュー

ツールバー 📝 で右下にメモパネルが開きます（複数行対応）。メモが付いているノードは右上に小さな青い点が表示されます。**AI 生成の子ノードはモデルの「why」で自動的にメモが埋まり、キャンバス上にインライン表示されるのでパネルを開く必要はありません。**

### リレーション 🔗 / サマリー 📎 / 境界 ⬚ / アウトライン 📋 / 検索

以前と同じ — 詳細はアプリ内ヘルプ（⌘?）を参照してください。

### レイアウト

ツールバー 🗺️ / 🌳 でトグル：**放射状**（デフォルト）/ **右ツリー**。

## ファイル形式

### JSON（無損失）

デフォルトのソース形式。すべてのデータを保持：構造、アイコン、色、メモ、サイズ、位置オフセット、リレーション、サマリー、境界。

`⌘S` またはツールバー 💾 で保存。

### Markdown（汎用）

階層をネスト箇条書きで表現。Obsidian / Typora / GitHub と互換。テキスト、アイコン、色、メモを保持。

```markdown
# 🎯 ルート <!-- c:#89b4fa -->

- 💡 ブランチ A <!-- c:#f9e2af -->
  > これはメモです
  - アイデア 1
- ⭐ ブランチ B
```

`⌘⇧E` またはツールバー 📤 で書き出し。

### OPML（XMind / MindNode / Logseq 互換）

標準 OPML 2.0。**XMind**、**MindNode**、**Logseq**、**iThoughts**、**Workflowy**、**OmniOutliner** にインポートして別ツールで編集を続けられます。

ツールバー 🌳 で書き出し。

### PDF（ベクター）

Electron の `printToPDF` で生成するネイティブベクター PDF（A3 横向き）。無限拡大可能、ラスタライズなし。

`⌘P` またはツールバー 📄 で書き出し。ブラウザ版は印刷ダイアログにフォールバック（「PDF として保存」）。

### 画像書き出し

- 🖼️ **SVG**：ベクター、無限拡大可、再編集可
- 📷 **PNG**：2× Retina 解像度

どちらもノード、エッジ、色、境界、リレーション、サマリーを含みます。

## 不具合報告

アプリ内：**Help → 报 Bug** で GitHub Issues 作成画面が開き、バージョン・OS・Electron バージョン・ローカルログファイルパスが事前入力されます。**Help → 打开日志文件夹** で `~/Library/Logs/Agentic Mindmap/` を直接開けます。

ログには各 AI 呼び出し（タイムスタンプ、レイテンシ、検出された種類、深さ）と全エラー（スタックトレース付き）が含まれます。

## 自動保存

ブラウザ版：変更ごとに `localStorage` へ書き込み、リロード後も維持。

デスクトップ版：同様。データは `~/Library/Application Support/Agentic Mindmap/Local Storage/` に保存されます。明示的にファイルへ書き出したい場合は `⌘S` で JSON エクスポート。

## リセット / クリア

- ツールバー ✨ 新規：現在のマップをクリア（確認ダイアログ付き）

## 既知の制約

- Markdown インポートは未知の構文を無視します
- SVG / PNG 書き出しに絵文字フォントは埋め込まれません（システムフォントのレンダリングに依存）
- 初回 `npm run dist` で Electron ビルド資産（~150 MB）をダウンロードするため時間がかかります
- DMG は未署名です。初回起動は `xattr -cr <app>` か右クリック → 開く で許可してください
